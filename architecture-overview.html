1 Core Engine - Entry & Query Loop

Core Engine은 Claude Code의 심장부로, 사용자가 claude 명령어를 실행하는 순간부터 Claude API와의 대화가 시작되기까지의 모든 초기화 과정을 담당합니다.

핵심 책임:

  • CLI 파싱: Commander.js를 사용해 --model, --resume, --print 등 50개 이상의 CLI 옵션을 파싱
  • 환경 설정: MDM(기업 관리) 설정, 키체인 토큰, 사용자 설정을 병렬로 로드하여 시작 시간 최적화
  • 상태 초기화: 전역 STATE 싱글톤과 React 기반 AppStateStore를 초기화하여 세션 전체에서 일관된 상태 관리
  • 쿼리 루프: 사용자 입력 → Claude API 호출 → 도구 실행 → 응답 렌더링의 무한 루프 관리
  • 세션 관리: 세션 ID 생성, 세션 전환, 비용 추적, 토큰 사용량 모니터링

main.tsx(4,683줄)가 진입점이며, setup.ts → bootstrap/state.ts → replLauncher.tsx 순서로 초기화가 진행됩니다.

main.tsx

main.tsx (4,683 lines)

역할: Claude Code CLI의 진입점. 모든 것이 여기서 시작됨.

  • main() - 메인 함수, Commander.js로 CLI 파싱
  • eagerLoadSettings() - --settings 플래그 조기 파싱

// useAppState - 선택자로 상태 슬라이스 구독
export function useAppState<T>(selector: (state: AppState) => T): T {
  const store = useAppStore()
  const get = () => {
    const state = store.getState()
    return selector(state)
  }
  return useSyncExternalStore(store.subscribe, get, get)
}

// useSetAppState - 업데이터만 반환 (구독 없음)
export function useSetAppState() {
  return useAppStore().setState
}

// useAppStateStore - 스토어 직접 접근
export function useAppStateStore(): AppStateStore {

핵심 책임:

  • 도구 정의: 각 도구는 ToolDef 인터페이스를 구현. name, description, inputSchema(Zod), call() 메서드, hasPermission() 권한 체크 포함
  • 파일 도구 (6개): Read(파일 읽기), Write(새 파일 생성), Edit(기존 파일 수정), Glob(파일 검색), Grep(내용 검색), NotebookEdit(Jupyter 편집)

  • 웹 도구 (2개): WebFetch(URL 콘텐츠 가져오기), WebSearch(웹 검색 수행) - HTML→Markdown 변환, 15분 캐시
  • 에이전트 도구: AgentTool(서브에이전트 생성), Task(백그라운드 태스크), TodoWrite(할일 관리) - 복잡한 작업을 분리 실행
  • 권한 시스템: 모든 도구 호출 전 hasPermission() 체크. 파일 경로, bash 명령 등에 대한 allow/deny/ask 규칙 적용
  • MCP 도구: Model Context Protocol을 통해 외부 서버가 제공하는 동적 도구 지원. GitHub, Slack, DB 등 무한 확장 가능

export const mcpServersCommand: Command = {
  name: 'mcp-servers',
  description: 'Show MCP server details and status',

  async call(args, context) {
    const servers = context.mcpClients

    // 각 서버의 상세 정보 수집
    const serverInfo = await Promise.all(
      servers.map(async (client) => ({
        name: client.serverName,
        status: client.state,  // 'connected' | 'disconnected' | 'error'
        transport: client.transportType,
        tools: await client.listTools(),
        resources: await client.listResources(),
        lastError: client.lastError
      }))
    )

    // 테이블 형태로 렌더링
    return {
      type: 'local',
      ui: <McpServersTable servers={serverInfo} />
    }
  }
}

export async function* withRetry<T>(
  fn: () => AsyncGenerator<T>,
  options: { querySource, signal, model, ... }
): AsyncGenerator<T | SystemAPIErrorMessage> {
  let retries = 0
  while (true) {
    try {
      yield* fn()
      return
    } catch (error) {
      if (error.status === 529 && shouldRetry529(querySource)) {
        // 용량 초과 - 지수 백오프로 재시도
        await sleep(BASE_DELAY_MS * Math.pow(2, retries))
        retries++
        continue
      }
      if (error.status === 401) {
        await handleOAuth401Error()  // 토큰 갱신 시도
        continue
      }
      throw error
    }
  }
}

  return {
    async start(command, args, options) {
      // 1. LSP 서버 프로세스 spawn

        env: { ...subprocessEnv(), ...options?.env }

      const reader = new StreamMessageReader(process.stdout)
      const writer = new StreamMessageWriter(process.stdin)
      connection = createMessageConnection(reader, writer)
      connection.listen()
    },

    async initialize(params) {
      const result = await connection.sendRequest('initialize', params)
      capabilities = result.capabilities
      await connection.sendNotification('initialized', {})
      isInitialized = true
      return result
    }
  }

export function createLSPServerManager(): LSPServerManager {
  // 클로저를 통한 private 상태
  const servers: Map<string, LSPServerInstance> = new Map()
  const extensionMap: Map<string, string[]> = new Map()
  const openedFiles: Map<string, string> = new Map()

  async function initialize(): Promise<void> {
    const { servers: serverConfigs } = await getAllLspServers()

    for (const [serverName, config] of Object.entries(serverConfigs)) {
      // 파일 확장자 → 서버 매핑 구축
      const fileExtensions = Object.keys(config.extensionToLanguage)
      for (const ext of fileExtensions) {
        extensionMap.set(ext.toLowerCase(), [...(extensionMap.get(ext) || []), serverName])
      }
      servers.set(serverName, createLSPServerInstance(serverName, config))
    }
  }

  function getServerForFile(filePath: string): LSPServerInstance | undefined {
    const ext = path.extname(filePath).toLowerCase()
    const serverNames = extensionMap.get(ext)
    return serverNames?.[0] ? servers.get(serverNames[0]) : undefined
  }

  async function openFile(filePath: string, content: string): Promise<void> {
    const server = await ensureServerStarted(filePath)
    if (!server) return

    const fileUri = pathToFileURL(path.resolve(filePath)).href
    if (openedFiles.get(fileUri) === server.name) return // 이미 열림

    await server.sendNotification('textDocument/didOpen', {
      textDocument: { uri: fileUri, languageId, version: 1, text: content }
    })
    openedFiles.set(fileUri, server.name)
  }

  return { initialize, shutdown, getServerForFile, ensureServerStarted,
           sendRequest, getAllServers, openFile, changeFile, saveFile,
           closeFile, isFileOpen }
}

6 Bridge/Remote - Remote Sessions

Bridge/Remote는 웹 브라우저(claude.ai/code)에서 로컬 터미널의 Claude Code를 원격으로 제어하는 시스템입니다. SSH 없이도 어디서든 자신의 개발 환경에 접근할 수 있게 해주는 핵심 인프라입니다.

핵심 책임:

  • Bridge 폴링 루프: claude --bridge로 실행하면 Anthropic 서버에 환경 등록 후, 주기적으로 작업을 폴링. WebSocket 대신 HTTP 폴링을 사용해 방화벽/프록시 환경에서도 안정적으로 동작

  • WebSocket 메시지: RemoteSessionManager가 WebSocket으로 서버의 실시간 이벤트(메시지, 권한 요청, 취소)를 수신. SDK 메시지는 onMessage 콜백으로 전달

  • 보안: JWT 토큰 자동 갱신(4시간마다), 신뢰 장치 토큰(X-Trusted-Device-Token), 서버 ID 검증(path traversal 방지)
  • Backoff 전략: 연결 실패 시 2초 → 2분까지 지수 백오프, 10분 후 포기. 시스템 슬립 감지로 불필요한 에러 방지

const settingsCache = new Map<SettingsSource, CachedSettings>()

interface CachedSettings {
  settings: SettingsJson
  errors: SettingsError[]
  lastModified: number
  checksum: string
}

function getCachedSettingsForSource(source: SettingsSource): CachedSettings | null {
  const cached = settingsCache.get(source)
  if (!cached) return null

  // 파일 변경 감지
  const currentMtime = statSync(getPathForSource(source)).mtimeMs
  if (currentMtime !== cached.lastModified) {
    // 캐시 무효화 - 다시 파싱 필요
    settingsCache.delete(source)
    return null
  }

  return cached
}

// 파싱 후 캐시 저장
function cacheSettings(source, settings, errors) {
  settingsCache.set(source, { settings, errors, lastModified: Date.now() })
}

type PermissionRule = {
  tool: string           // 'Bash', 'Edit', '*'
  path?: string          // 'src/**', '/etc/*' (glob)
  command?: string       // 'npm *', 'git *'
  decision: 'allow' | 'deny' | 'ask'
}

function checkPermissionRules(tool, input): PermissionDecision {
  const rules = getEffectivePermissionRules()  // user + project + managed

  // 가장 구체적인 규칙 먼저 (specificity 순)
  for (const rule of sortBySpecificity(rules)) {
    if (matchesRule(tool, input, rule)) {
      return {
        decision: rule.decision,
        rule: rule,
        reason: `Matched rule: ${rule.tool} ${rule.path || rule.command}`
      }
    }
  }

  // 기본값: 사용자에게 물어봄
  return { decision: 'ask', reason: 'No matching rule' }
}

// 범용 에러 메시지 추출
export function errorMessage(error: unknown): string {
  if (error === null || error === undefined) {
    return 'Unknown error'
  }

  if (typeof error === 'string') {
    return error
  }

  if (error instanceof Error) {
    // 중첩된 cause 처리
    if (error.cause) {
      return `${error.message}: ${errorMessage(error.cause)}`
    }
    return error.message
  }

  // API 에러 등 특수 객체
  if (typeof error === 'object' && 'message' in error) {
    return String(error.message)
  }

  // 최후 수단
  try {

// 모션을 목표 커서 위치로 해석 (순수 함수)
function resolveMotion(key: string, cursor: Cursor, count: number): Cursor {
  let result = cursor
  for (let i = 0; i < count; i++) {
    const next = applySingleMotion(key, result)
    if (next.equals(result)) break  // 더 이상 이동 불가
    result = next
  }
  return result
}

function applySingleMotion(key: string, cursor: Cursor): Cursor {
  switch (key) {
    case 'h': return cursor.left()
    case 'l': return cursor.right()
    case 'j': return cursor.downLogicalLine()
    case 'k': return cursor.upLogicalLine()
    case 'w': return cursor.nextVimWord()
    case 'b': return cursor.prevVimWord()
    case '0': return cursor.startOfLogicalLine()
    case '$': return cursor.endOfLogicalLine()
    ...
  }
}
